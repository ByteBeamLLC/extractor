# Getlate's Tool Page Strategy

## Overview
Mickey (Getlate founder) builds free tools as lead magnets targeting high-intent keywords. Tools rank organically on Google AND serve as landing pages for paid ads. Zero to $40K MRR in 7 months using this playbook.

## Core Principle
Free tool → immediate value → trust → pipeline to paid product. The tool IS the content. Every section serves the funnel.

---

# Part 1: Section Order

Every tool page follows this sequence:

### 1. Hero + Tool Input
- Tool label badge above title
- Big bold title (action-oriented)
- One-line subtitle
- Social proof directly under title: ★ 4.8 (X reviews) · Trusted by thousands
- **The tool itself** — input field / upload area right in the hero
- Tiny trust line below tool: processing details

### 2. API / Product Upsell (EARLY — right after tool)
- Boxed callout with code icon
- Heading: "Want to [do this thing] via API?"
- Brief pitch + two CTAs: "Explore API →" and "Docs"
- **Position is critical** — catches power users before they scroll away

### 3. Cross-Sell Links
- One-liner with inline links to related paid products, comparison pages, blog
- Tone: helpful suggestion, not hard sell

### 4. Why Use This Tool (Benefits)
- 3 cards in a row: icon + bold title + 1-2 sentence description
- Benefit-driven, not feature-driven

### 5. How It Works
- 3 numbered steps with "Important notes" callout below

### 6. Educational Content (varies by tool)
- Domain-specific guides, tips, do's/don'ts, comparisons
- **Purpose**: SEO long-tail keywords + demonstrates expertise

### 7. Perfect For (User Personas)
- 4 persona cards with brief descriptions
- Helps visitors self-identify → increases conversion

### 8. FAQ
- 4-10 questions in grid layout
- Targets long-tail SEO queries + handles objections

### 9. Social Proof / Metrics
- Stats strip with quantified confidence builders

### 10. Works Everywhere (Compatibility)
- Device/browser grid: Desktop, Mobile, Tablet

### 11. More Free Tools
- 3 cross-link cards to other tools in the ecosystem

### 12. Final Product CTA
- Aspirational heading + brief pitch + two buttons (primary + secondary)

# Part 2: Content Strategy

## Hero Copy Formulas

### Title Pattern
Action verb + object. No filler. No "tool" or "converter" in the h1 — save that for the badge.

| Page | Title |
|------|-------|
| TikTok Downloader | "Download TikTok Videos" |
| YouTube Downloader | "Download YouTube Videos" |
| YouTube Audio | "Extract Audio from YouTube" |
| YouTube Shorts | "Download YouTube Shorts" |
| Instagram Handle | "Check Instagram Username Availability" |
| Hashtag Checker | "Instagram Banned Hashtag Checker" |
| YouTube Live | "YouTube Live Downloader" |

**Pattern**: `[Action verb] [Platform] [Content type]`
**Parsli equivalent**: "Convert PDF to Excel"

### Subtitle Pattern
Short, comma-separated value props. Always includes "free" and "no sign-up".

| Variation | Used on |
|-----------|---------|
| "Free, simple, no sign-up" | TikTok, YouTube Video, YouTube Audio |
| "Free checker + complete naming guide" | Instagram Handle |
| "Free HD downloads, no sign-up" | YouTube Shorts, Reels |
| "Download live streams and broadcasts in 720p, 1080p or 4K" | YouTube Live |

**Formula**: `Free, [key differentiator], no sign-up`

### Description Pattern (under subtitle)
One sentence explaining what the tool does + one sentence about how. Ends with a use-case or disclaimer.

**Template**: "Paste any [platform] URL and choose from available [formats/options]. See [preview type] and select your preferred [quality/format]. [Use case or disclaimer]."

### Social Proof Line
Identical across almost all pages (consistency = credibility):
- "★ 4.8 (2,847 reviews) · Trusted by thousands"
- YouTube Live uses higher number: "4.8 (12,500+ reviews)"

**Key insight**: Same rating everywhere builds brand-level trust, not tool-level trust.

## Cross-Sell Copy

Single sentence. Identical across all pages. Reads as a helpful aside, not a pitch.

**Getlate's exact copy**: "Building a social media app? Try our Social Media API for Developers — compare the best social media APIs or see how Late stacks up against Hootsuite and Buffer."

**Structure**: `[Qualifying question]? Try [product] — [comparison link] or see how [brand] stacks up against [competitor 1] and [competitor 2].`

**Links embedded**: product page, comparison/alternatives page, competitor pages

**Parsli equivalent**: "Processing invoices or bank statements at scale? Invoice Parsing and Bank Statement Extraction are built for that. Or see how Parsli compares to Docparser and Nanonets."

## "Why Use This Tool" Cards

Always exactly 3 cards. Each has: icon + bold 2-3 word title + 1 sentence description.

**The 3 card themes rotate around these pillars:**

| Pillar | Example titles | Description pattern |
|--------|---------------|---------------------|
| **Unique capability** | "No watermark option", "Shorts optimized", "High quality audio", "Live stream capture" | What makes this tool specifically good at its job |
| **Zero friction** | "No signup", "No signup required" | "Use it instantly in your browser. We stream files to you and don't store your downloads." |
| **Free + simple** | "Free & simple", "Instant download", "Fast extraction" | "Paste a link, choose format, download. Works on desktop and mobile." OR "No ads, no watermarks, no registration." |

**Pattern**: Card 1 = tool-specific value prop. Card 2 = no signup. Card 3 = free/fast/simple.

**Copy style**: Descriptions are 1 sentence max. Benefit-first. Never start with "Our tool..." or "We provide...". Start with action or outcome.

---

## "How It Works" Steps

Always exactly 3 steps. Each: bold title + 1 sentence description.

**Step formula (consistent across all pages)**:
1. **Input step**: "Copy/paste [URL/file]" — describes what to provide
2. **Processing step**: "Choose/See [format/preview]" — shows what happens
3. **Output step**: "Download [result]" — the payoff

**Important notes callout** always follows. Lists 2-4 caveats/limitations. Honest about what doesn't work — this builds trust AND funnels to the paid product.

**Notes always include**:
- What content types might fail (private, restricted, DRM)
- Technical limitation acknowledgment
- "Use responsibly" / respect creator rights
- For Parsli: "Works best with text-based PDFs. Scanned documents need OCR — try Parsli AI."

## Educational Content Patterns

This is where pages diverge the most. Content is tool-specific but follows clear patterns:

### Type A: "Supported Formats" (Downloader tools)
Structured as 3 sub-sections with bullet lists:
- **Format 1** (e.g., Video Formats): 4-5 bullets about codec, compatibility
- **Format 2** (e.g., Audio Formats): 4-5 bullets
- **Quality Options**: 4-5 bullets about resolution levels

### Type B: "Do This / Avoid This" (Checker tools)
Two-column layout:
- **Do This**: 5-6 actionable best practices
- **Avoid This**: 5-6 specific mistakes to avoid
- Copy is imperative: "Use 3-5 highly relevant hashtags" not "You should use..."

### Type C: "Deep Guide" (Complex tools like Instagram Handle Checker)
Multiple subsections, each a mini-article:
- "Why a Great Instagram Handle Matters" (4 reasons with bold keyword + explanation)
- "Naming Formulas You Can Steal" (template patterns)
- "Smart Variations If Your Name Is Taken" (6 alternatives)
- "Good vs Bad Examples" (side-by-side with annotations)
- "Cross-Platform Checklist" (action list)
- "Troubleshooting" (common issues)

**Key insight**: The handle checker page is 3-4x longer than downloader pages. Tools with more educational SEO opportunity get more content. Tools that are straightforward (downloaders) keep it concise.

### Type D: "Comparison Tables" (when competitors/alternatives exist)
Table format comparing the tool to alternatives or formats:
- YouTube Audio: "M4A vs MP3" comparison table
- YouTube Shorts: "Shorts vs Regular Videos" comparison
- YouTube Reels: "Reels vs Instagram Reels vs Regular Videos"

**Parsli equivalent**: "Free Tool vs Parsli AI" comparison table (already have this).

### Content volume per tool type:
| Tool complexity | Educational sections | FAQ count |
|----------------|---------------------|-----------|
| Simple (downloaders) | 1-2 sections | 4-6 |
| Medium (audio extractor) | 2-3 sections + comparison table | 4 |
| Complex (handle checker, hashtag checker) | 5-8 sections, deep guides | 9-10 |

## FAQ Content Strategy

### Question formula patterns:
1. **"Is this [thing] free?"** — Always first or second. Answer: "Yes, completely free! No sign-ups, no limits, no hidden fees."
2. **"Do you store my files?"** — Privacy reassurance. Answer: "No. Files are [processed/streamed] directly to your browser..."
3. **"What [types/quality] work?"** — Sets expectations. Honest about limitations.
4. **"Why do some [things] fail?"** — Acknowledges failures gracefully, names specific reasons.
5. **"What's the difference between [free tool] and [paid product]?"** — Natural upsell. Lists what paid product adds.
6. **Long-tail SEO questions** — "How to download YouTube videos without premium?", "What is an Instagram shadowban?" — written for Google Featured Snippets.

### Answer copy style:
- Start with a direct answer: "Yes!", "No.", "Most [things] work, but..."
- Keep under 3 sentences for simple questions
- For SEO questions: longer answers (4-5 sentences) that could rank as Featured Snippets
- Tone: confident, helpful, specific
- Subtly mention paid product in 1-2 answers as a natural solution

### FAQ count by tool complexity:
- Simple tools (downloaders): 4-6 questions
- Medium tools: 6-8 questions
- Complex tools (checkers with education): 9-10 questions

# Part 3: Cross-Cutting Content Patterns

## Copy Style Rules

1. **Conversational, not corporate** — "Just paste a link" not "Simply input the URL into the designated field"
2. **Specific metrics over vague claims** — "250K+ downloads" not "Thousands of downloads"
3. **Benefit-first language** — "Download videos" not "Video downloading tool"
4. **Short sentences** — Most descriptions are 1 sentence. Max 2.
5. **No jargon** — "Works on all devices" not "Cross-platform compatible"
6. **Active voice** — "We stream files to you" not "Files are streamed"
7. **Second person** — "Your files never leave your device" not "Users' files..."
8. **Imperative for actions** — "Copy the URL" not "You should copy the URL"

## Tone Progression Down the Page

| Position | Tone | Example |
|----------|------|---------|
| Hero | Direct, confident, backed by proof | "Download TikTok Videos — Free, simple, no sign-up — ★ 4.8" |
| API upsell | Technical, professional | "Integrate into your workflow. Build automations, batch process." |
| Cross-sell | Casual suggestion | "Building a social media app? Try our..." |
| Benefits | Punchy, benefit-first | "No watermark option — Get clean MP4 downloads" |
| How it works | Clear, instructional | "Copy URL → Choose format → Download" |
| Educational | Expert, generous | Deep guides, best practices, comparisons |
| FAQ | Helpful, honest | "Yes, completely free!" / "Some videos may not be available" |
| Final CTA | Aspirational | "Grow faster with Late" |

## Consistency Rules (same across ALL pages)

These elements are identical across every tool page:
- Social proof rating and text
- Cross-sell one-liner
- API upsell structure
- "Works Everywhere" device list
- Final CTA copy
- "Use Responsibly" disclaimer
- Footer links

**Why**: Brand consistency. Users who visit multiple tools see the same trust signals. Reduces cognitive load.
